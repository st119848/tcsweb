Java Programming Language: พื้นฐานและแนวคิด

Java เป็นภาษาโปรแกรมที่ถูกออกแบบมาให้ใช้งานง่ายและปลอดภัย เหมาะสำหรับผู้เริ่มต้นและมืออาชีพ

Java คืออะไร?

Java เป็นภาษาโปรแกรมเชิงวัตถุ (OOP) ที่เน้นการนำกลับมาใช้ใหม่และการจัดการโค้ดให้เป็นระบบ

จุดเด่นของ Java

  • Write Once, Run Anywhere (WORA): เขียนโค้ดครั้งเดียว รันได้ทุกระบบ
  • ปลอดภัย: มีระบบจัดการหน่วยความจำอัตโนมัติ
  • มีชุมชนขนาดใหญ่และเครื่องมือช่วยเหลือมากมาย

Java Virtual Machine (JVM)

JVM คือโปรแกรมที่ช่วยให้โค้ด Java สามารถรันบนระบบปฏิบัติการต่างๆ ได้โดยไม่ต้องแก้ไขโค้ด

โครงสร้างโปรแกรม Java เบื้องต้น

public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, world!");
    }
}

ทุกโปรแกรม Java จะมีเมธอด main เป็นจุดเริ่มต้น

ตัวแปรและชนิดข้อมูล

  • int: ตัวเลขจำนวนเต็ม
  • double: ตัวเลขทศนิยม
  • String: ข้อความ
  • boolean: จริง/เท็จ
int age = 20;
double price = 99.99;
String name = "Alice";
boolean isStudent = true;

การรับค่าและแสดงผล

import java.util.Scanner;
Scanner sc = new Scanner(System.in);
System.out.print("กรุณากรอกชื่อ: ");
String name = sc.nextLine();
System.out.println("สวัสดี " + name);

เงื่อนไข (if-else)

int score = 80;
if (score >= 50) {
    System.out.println("สอบผ่าน");
} else {
    System.out.println("สอบตก");
}

การวนลูป (for, while)

for (int i = 1; i <= 5; i++) {
    System.out.println(i);
}

int i = 1;
while (i <= 5) {
    System.out.println(i);
    i++;
}

ฟังก์ชัน (Method)

public static int add(int a, int b) {
    return a + b;
}
System.out.println(add(3, 5)); // 8

คลาสและอ็อบเจกต์ (Class & Object)

public class Person {
    String name;
    int age;
    public void sayHello() {
        System.out.println("Hello " + name);
    }
}
Person p = new Person();
p.name = "Bob";
p.sayHello();

การสืบทอด (Inheritance)

class Animal {
    void eat() { System.out.println("กินอาหาร"); }
}
class Dog extends Animal {
    void bark() { System.out.println("เห่า"); }
}
Dog d = new Dog();
d.eat();
d.bark();

Polymorphism (พหุรูป)

Animal a = new Dog();
a.eat(); // เรียก eat() ของ Dog ได้

Polymorphism คือความสามารถในการใช้คลาสลูกแทนคลาสแม่ได้

Encapsulation (การห่อหุ้มข้อมูล)

class Account {
    private double balance;
    public void deposit(double amount) { balance += amount; }
    public double getBalance() { return balance; }
}

Encapsulation คือการซ่อนข้อมูลภายในคลาสและเข้าถึงผ่านเมธอด

Exception Handling (การจัดการข้อผิดพลาด)

try {
    int x = 10 / 0;
} catch (ArithmeticException e) {
    System.out.println("เกิดข้อผิดพลาด: " + e.getMessage());
}

การใช้งาน Library และ Framework

Java มีไลบรารีและเฟรมเวิร์กมากมาย เช่น Spring, Hibernate, Gson, Apache Commons

Spring Boot: พื้นฐานและแนวคิด

Spring Boot คือเฟรมเวิร์กที่ช่วยให้การสร้างแอปพลิเคชัน Java เป็นเรื่องง่ายและรวดเร็ว

Spring Boot คืออะไร?

Spring Boot สร้างบน Spring Framework เพื่อช่วยลดขั้นตอนการตั้งค่าและเริ่มต้นโปรเจกต์

ข้อดีของ Spring Boot

  • ตั้งค่าอัตโนมัติ (Auto-Configuration)
  • มี Starter Dependencies ให้เลือกใช้
  • เหมาะสำหรับ Production
  • มี Community ใหญ่และเอกสารครบถ้วน

Starter Dependencies คืออะไร?

Starter Dependencies คือชุดไลบรารีที่รวมทุกอย่างที่จำเป็นสำหรับงานแต่ละประเภท เช่น spring-boot-starter-web สำหรับสร้าง Web API

Auto-Configuration ทำงานอย่างไร?

Spring Boot จะตรวจสอบ dependencies ที่เราเพิ่มเข้าไป แล้วตั้งค่าระบบให้โดยอัตโนมัติ เช่น ถ้าเพิ่ม spring-boot-starter-data-jpa จะตั้งค่าฐานข้อมูลให้เอง

โครงสร้างโปรเจกต์ Spring Boot

src/
  main/
    java/
      com/example/demo/
        DemoApplication.java
    resources/
      application.properties

สร้างโปรเจกต์ Spring Boot ด้วย Spring Initializr

Spring Initializr คือเว็บไซต์ที่ช่วยสร้างโปรเจกต์ Spring Boot ได้ง่ายๆ

https://start.spring.io/

ไฟล์ application.properties

ใช้สำหรับตั้งค่าต่างๆ เช่น พอร์ต, ฐานข้อมูล, การเชื่อมต่อ API

server.port=8080
spring.datasource.url=jdbc:mysql://localhost:3306/demo

สร้าง RESTful API ด้วย Spring Boot

@RestController
@RequestMapping("/api/tasks")
public class TaskController {
    @GetMapping
    public List getAllTasks() { return taskRepository.findAll(); }
    @PostMapping
    public Task createTask(@RequestBody Task task) { return taskRepository.save(task); }
}

Annotation สำคัญใน Spring Boot

  • @RestController: สำหรับสร้าง REST API
  • @RequestMapping: กำหนด URL หลัก
  • @GetMapping, @PostMapping, @PutMapping, @DeleteMapping: กำหนด HTTP Method
  • @Autowired: ฉีด dependencies อัตโนมัติ

การเชื่อมต่อฐานข้อมูลกับ Spring Boot

Spring Boot รองรับฐานข้อมูลหลายประเภท เช่น MySQL, PostgreSQL, MongoDB

spring.datasource.url=jdbc:mysql://localhost:3306/demo
spring.datasource.username=root
spring.datasource.password=1234

Spring Boot กับ JPA/Hibernate

Spring Boot ใช้ Spring Data JPA และ Hibernate ในการจัดการข้อมูลแบบ ORM

@Entity
public class Task {
    @Id
    @GeneratedValue
    private Long id;
    private String name;
}

การทดสอบ (Testing) ใน Spring Boot

  • Unit Test: ทดสอบเมธอดแต่ละตัว
  • Integration Test: ทดสอบการทำงานร่วมกันของหลายส่วน
  • ใช้ JUnit, Mockito, Spring Boot Test

Deploy Spring Boot ไปยัง Production

  • สามารถรันเป็นไฟล์ JAR หรือ WAR
  • Deploy บน Cloud เช่น AWS, Heroku, Google Cloud
  • รองรับ Docker และ Kubernetes

Spring Boot Actuator

Actuator ให้ endpoint สำหรับตรวจสอบสุขภาพระบบ เช่น /actuator/health

Spring Boot Security

Spring Boot Security ใช้สำหรับจัดการ Authentication และ Authorization

@EnableWebSecurity
public class SecurityConfig extends WebSecurityConfigurerAdapter {
    @Override
    protected void configure(HttpSecurity http) throws Exception {
        http.authorizeRequests().anyRequest().authenticated();
    }
}

Spring Boot: การสร้าง Web API และ RESTful Services

  • Starter Dependencies: เช่น spring-boot-starter-web
  • Auto-Configuration: ตั้งค่าอัตโนมัติตาม dependencies

RESTful API

  • GET: ดึงข้อมูล
  • POST: สร้างข้อมูลใหม่
  • PUT/PATCH: อัปเดตข้อมูล
  • DELETE: ลบข้อมูล
@RestController
@RequestMapping("/api/tasks")
public class TaskController {
    @GetMapping
    public List getAllTasks() { return taskRepository.findAll(); }
    @PostMapping
    public Task createTask(@RequestBody Task task) { return taskRepository.save(task); }
}

Dependency Management: Maven & Gradle

Maven และ Gradle คือเครื่องมือจัดการโปรเจกต์ Java ที่ช่วยให้การพัฒนาเป็นระบบและอัตโนมัติ

Dependency คืออะไร?

Dependency คือไลบรารีหรือเฟรมเวิร์กที่โปรเจกต์ของเราต้องใช้ เช่น Spring Boot, JUnit, MySQL Driver

ปัญหาถ้าไม่มี Dependency Management

  • ต้องดาวน์โหลดไฟล์ JAR เอง
  • เสี่ยงต่อการใช้เวอร์ชันผิดหรือขาดไฟล์
  • จัดการอัปเดตยาก

Maven คืออะไร?

Maven ใช้ไฟล์ pom.xml ในการกำหนด dependencies และขั้นตอน build

โครงสร้างไฟล์ pom.xml

<project>
  <dependencies>
    <dependency>
      <groupId>org.springframework.boot</groupId>
      <artifactId>spring-boot-starter-web</artifactId>
    </dependency>
  </dependencies>
</project>

Gradle คืออะไร?

Gradle ใช้ไฟล์ build.gradle ที่เขียนด้วย Groovy หรือ Kotlin DSL

โครงสร้างไฟล์ build.gradle

plugins {
    id 'org.springframework.boot' version '3.0.0'
}
dependencies {
    implementation 'org.springframework.boot:spring-boot-starter-web'
}

เปรียบเทียบ Maven กับ Gradle

หัวข้อMavenGradle
ไฟล์ตั้งค่าpom.xml (XML)build.gradle (Groovy/Kotlin)
ความยืดหยุ่นน้อยสูง
ความเร็วช้าเร็วกว่า
Communityใหญ่กำลังเติบโต

การเพิ่ม Dependency ใน Maven

<dependency>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter-data-jpa</artifactId>
</dependency>

การเพิ่ม Dependency ใน Gradle

implementation 'org.springframework.boot:spring-boot-starter-data-jpa'

Build Automation คืออะไร?

Build Automation คือการคอมไพล์ ทดสอบ และแพ็กเกจโปรเจกต์โดยอัตโนมัติ

คำสั่งพื้นฐาน Maven

  • mvn clean: ลบไฟล์ build เก่า
  • mvn compile: คอมไพล์โค้ด
  • mvn test: รัน unit test
  • mvn package: สร้างไฟล์ JAR/WAR

คำสั่งพื้นฐาน Gradle

  • gradle clean
  • gradle build
  • gradle test

การจัดการเวอร์ชัน Dependency

สามารถระบุเวอร์ชันที่ต้องการในไฟล์ตั้งค่า เช่น spring-boot-starter-web:3.0.0

Repository คืออะไร?

Repository คือแหล่งเก็บไลบรารี เช่น Maven Central, JCenter

การแก้ปัญหา Dependency Conflict

ถ้า dependency หลายตัวใช้เวอร์ชันต่างกัน อาจเกิดปัญหา ต้องจัดการด้วย exclusion หรือ resolution strategy

การใช้งานกับ Spring Boot

Spring Boot รองรับทั้ง Maven และ Gradle สามารถเลือกใช้ได้ตามความถนัด

สรุป

  • Maven และ Gradle ช่วยให้การพัฒนา Java เป็นระบบและง่ายขึ้น
  • เลือกใช้ตามความเหมาะสมของโปรเจกต์
