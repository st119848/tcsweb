Maven / Gradle

การจัดการ Project และ Build Automation

บทนำ Build Tools

Build Tools คือเครื่องมือที่ช่วยจัดการกระบวนการ build ของโปรเจกต์ เช่น คอมไพล์โค้ด, รัน test, แพ็กเกจ

ปัญหาแบบเดิม

  • ต้องคอมไพล์โค้ดเองด้วย javac
  • จัดการ dependencies เอง
  • ไม่มีมาตรฐานในการ build

Maven คืออะไร?

Maven เป็น build tool ที่ใช้ไฟล์ pom.xml ในการกำหนด dependencies และขั้นตอน build

โครงสร้างโปรเจกต์ Maven

src/
  main/
    java/
    resources/
  test/
    java/
pom.xml

ไฟล์ pom.xml

<project>
  <modelVersion>4.0.0</modelVersion>
  <groupId>com.example</groupId>
  <artifactId>demo</artifactId>
  <version>1.0.0</version>
  <dependencies>
    <dependency>
      <groupId>org.springframework.boot</groupId>
      <artifactId>spring-boot-starter-web</artifactId>
    </dependency>
  </dependencies>
</project>

Gradle คืออะไร?

Gradle เป็น build tool ที่ใช้ไฟล์ build.gradle และมี DSL ที่ยืดหยุ่นกว่า Maven

โครงสร้างโปรเจกต์ Gradle

src/
  main/
    java/
    resources/
  test/
    java/
build.gradle

ไฟล์ build.gradle

plugins {
    id 'java'
}
group = 'com.example'
version = '1.0.0'
repositories {
    mavenCentral()
}
dependencies {
    implementation 'org.springframework.boot:spring-boot-starter-web'
}

Dependency Management ใน Maven

<dependency>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter-data-jpa</artifactId>
  <version>3.0.0</version>
</dependency>

Dependency Management ใน Gradle

dependencies {
    implementation 'org.springframework.boot:spring-boot-starter-data-jpa:3.0.0'
}

Build Lifecycle ใน Maven

  • validate: ตรวจสอบโปรเจกต์
  • compile: คอมไพล์โค้ด
  • test: รัน unit test
  • package: แพ็กเกจเป็น JAR/WAR
  • install: ติดตั้งใน local repository
  • deploy: deploy ไป remote repository

Build Lifecycle ใน Gradle

  • compileJava: คอมไพล์ Java
  • processResources: จัดการ resources
  • test: รัน test
  • jar: สร้าง JAR
  • publish: publish ไป repository

คำสั่งพื้นฐาน Maven

  • mvn clean: ลบไฟล์ build
  • mvn compile: คอมไพล์
  • mvn test: รัน test
  • mvn package: แพ็กเกจ
  • mvn install: ติดตั้งใน local repo

คำสั่งพื้นฐาน Gradle

  • gradle clean
  • gradle build
  • gradle test
  • gradle jar

การจัดการ Multi-Module ใน Maven

<modules>
  <module>core</module>
  <module>web</module>
</modules>

การจัดการ Multi-Module ใน Gradle

include 'core', 'web'

Plugin ใน Maven

<build>
  <plugins>
    <plugin>
      <groupId>org.springframework.boot</groupId>
      <artifactId>spring-boot-maven-plugin</artifactId>
    </plugin>
  </plugins>
</build>

Plugin ใน Gradle

plugins {
    id 'org.springframework.boot' version '3.0.0'
}

การจัดการ Repository

  • Maven Central: repository หลัก
  • JCenter: repository อื่น
  • Private Repository: สำหรับองค์กร

การแก้ปัญหา Dependency Conflict

  • Maven: ใช้ exclusion หรือ dependencyManagement
  • Gradle: ใช้ resolution strategy

สรุป

  • Maven และ Gradle ช่วยให้การพัฒนา Java เป็นระบบและอัตโนมัติ
  • เลือกใช้ตามความเหมาะสมของโปรเจกต์
